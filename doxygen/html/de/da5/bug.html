Member tesseract_planning::totg::Trajectory::Trajectory (const Path &path, const Eigen::VectorXd &max_velocity, const Eigen::VectorXd &max_acceleration, double time_step=0.001)
A negative path velocity error will occur if the path contains points that are close to but not equal to the start or end point. A workaround (implemented in TimeOptimalTrajectoryGeneration::computeTimesteps) is to append an extra joint that increments such that no point is ever repeated. See https://github.com/ros-industrial-consortium/tesseract_planning/issues/27
