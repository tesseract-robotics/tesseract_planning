1) Clone repository into your workspace 2) Clone the repositories in the dependencies.\+rosinstall file using wstool or some other method (e.\+g. manually git cloning them) 3) Build the workspace using catkin tools, colcon, or a similar tool

NOTE\+: To speed up clean build you may want to add tesseract\+\_\+ext to an extended workspace.

NOTE\+: Install Task\+Flow from \href{https://launchpad.net/~ros-industrial/+archive/ubuntu/ppa}{\texttt{ ROS-\/\+Industrial PPA}}.\hypertarget{index_autotoc_md8}{}\doxysubsection{Building with Clang-\/\+Tidy Enabled}\label{index_autotoc_md8}
Must pass the -\/DTESSERACT\+\_\+\+ENABLE\+\_\+\+CLANG\+\_\+\+TIDY=ON to cmake when building. This is automatically enabled if cmake argument -\/DTESSERACT\+\_\+\+ENABLE\+\_\+\+TESTING\+\_\+\+ALL=ON is passed.\hypertarget{index_autotoc_md9}{}\doxysubsection{Building Tests}\label{index_autotoc_md9}
Must pass the -\/DTESSERACT\+\_\+\+ENABLE\+\_\+\+TESTING=ON to cmake when wanting to build tests. This is automatically enabled if cmake argument -\/DTESSERACT\+\_\+\+ENABLE\+\_\+\+TESTING\+\_\+\+ALL=ON is passed.

NOTE\+: If you are building using catkin tools, use {\ttfamily catkin build -\/-\/force-\/cmake -\/DTESSERACT\+\_\+\+ENABLE\+\_\+\+TESTING=ON}.\hypertarget{index_autotoc_md10}{}\doxysubsubsection{Running Tests}\label{index_autotoc_md10}
Tesseract packages use ctest because it is ROS agnostic, so to run the test call {\ttfamily catkin test -\/-\/no-\/deps tesseract\+\_\+motion\+\_\+planners tesseract\+\_\+process\+\_\+managers tesseract\+\_\+time\+\_\+parameterization}\hypertarget{index_autotoc_md11}{}\doxysubsection{Building Code Coverage}\label{index_autotoc_md11}
Must pass the -\/DTESSERACT\+\_\+\+ENABLE\+\_\+\+CODE\+\_\+\+COVERAGE=ON to cmake when wanting to build code coverage. The code coverage report is located in each individuals build directory inside a ccov/all-\/merged folder. Open the index.\+html file to see the packages code coverage report.
